# Design Tokens

Design tokens are the visual design atoms of the Ozean Licht design system—the named entities that store visual design attributes. They ensure consistency across all applications in the ecosystem.

## What Are Design Tokens?

Design tokens are platform-agnostic variables that define the visual characteristics of your application:

- **Colors**: Brand colors, semantic colors, state colors
- **Typography**: Font families, sizes, weights, line heights
- **Spacing**: Margins, padding, gaps
- **Shadows**: Elevation and depth
- **Borders**: Radii, widths, styles
- **Animations**: Durations, easings, transitions

## Using Tokens in Components

### CSS Variables

```css
.my-component {
  color: hsl(var(--foreground));
  background: hsl(var(--background));
  padding: var(--spacing-4);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-md);
}
```

### Tailwind Classes

Tokens are mapped to Tailwind utilities:

```tsx
<div className="bg-background text-foreground p-4 rounded-lg shadow-md">
  Content
</div>
```

### TypeScript

Access tokens programmatically:

```typescript
const tokens = {
  colors: {
    primary: 'hsl(210, 100%, 50%)',
    destructive: 'hsl(0, 84.2%, 60.2%)',
  },
  spacing: {
    4: '1rem',
    8: '2rem',
  },
};
```

## Best Practices

### Do's ✅

- Use design tokens instead of hard-coded values
- Leverage semantic color names (`destructive` over `red-500`)
- Use spacing scale consistently (multiples of 4)
- Apply elevation system for depth
- Follow brand guidelines (turquoise + cosmic theme)

### Don'ts ❌

- Don't hard-code colors or spacing
- Don't create custom shadows outside the scale
- Don't bypass the token system with inline styles
- Don't use non-standard font weights
- Don't ignore dark mode variants

## Token Updates

Design tokens are centralized in:

- `shared/ui-components/src/styles/globals.css` - CSS variables
- `tailwind.config.js` - Tailwind theme extension

Updates to tokens automatically propagate to all components.
